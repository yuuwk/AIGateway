观察：

- **首轮（id=2）**：`cache_read=0`，属冷启动，22,195 token 全部计入 input。
- **后续主对话**：每轮仅 **48–407** 新增 input，同时 **57,856–60,672** token 从缓存读取。
- **`cache_read` 随对话递增**（57,856 → 60,672），说明缓存在持续累积复用。
